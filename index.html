Arch Install Guide

Section 1.0: Network

Before continuing throught this setup check that you have to do this.

$ ping 1.1.1.1

If you get something along the lines off

$ 64 bytes from 1.1.1.1: icmp_seq=1 ttl=57 time=18.7 ms

then you have a working network connection and should scroll to Section 2.0.

If not no worry's this is a simple fix in the install procces.

Section 1.1: Setting up a temporary connection

First off run this following command

$ iwctl

This will enter into iwctl. Next run.

$ device list

This will display your wifi interface's. Next run.

$ station (interface) scan

This will start a scan for networks. Next run.

$ station (interface) scan

Now to get the names of availbe networks run the command below.

$ station (interface) get-networks

Now it will prompt you for a keyphrase enter your wifi password.

This will connect to the network you want to connect to. Next run.

$ exit

That will exit iwctl

Section 1.2: Testing Connection

$ ping 1.1.1.1

If you get something along the lines off

$ 64 bytes from 1.1.1.1: icmp_seq=1 ttl=57 time=18.7 ms

then you have a working network connection and should scroll to Section 2.0 if not go back to 1.1.

Section 2.0: Setting up the system

Now that you have a working network connection you can start setting up the system.

First off run this following command

$ archinstall

This will enter into archinstall.

Now you should be shown a list of numbers and intials of countrys.

Now to select a country enter the number that matches your country.

Example for the UK.

$ 25

Now you should be shown a list of numbers and languages.

Now to select a language enter the number that matches your language.

Example for the UK.

$ 54

Now you should be shown a list of numbers and diffrent drives.

Now to select a drive enter the number that matches your drive.

Example for my Computer(sda).

$ 1

Next thing it will ask you about is what type of filesystem you would like.

I personlly use and recommend ext4.

$ 1

It will know prompt you for a disk encyrption password.

I choose to leave this blank thought I wouldnt recommend this though.

Next it will ask you to choose your hostname.

$ ciaran

Next it will ask you to choose a root password.

now dont be allarmed if it seems as if ure not typing this is something that linux does when entering passwords into things like sudo.

Next retype this to confirm your root password.

Section 2.1: Users

In this section you will create users

Things you should understand

  • What sudo/sudoers is.
  • Who is going to use each account.
  • Why should someone not have sudo.

Now to create a user enter the selected name for it.

Example:

$ ciaran

Now to create a password for the user enter the password.

now remeber that even if it seems ure not typing this is something that linux does when entering passwords into things like sudo.

Next retype this to confirm your password.

Now it will ask if this user should be a sudoer.

If you dont know what sudoers is you should read this

Or if your lazy just make your own acount and give it sudoers. To make this choice type y or n (yes:no).

$ y

If you are only making one user just press enter to skip if not repeat that step again as required.

Next it will prompt you about what type of profiles ,window mangers/desktop enviroments.

But we will enter 1 for minimal

$ 1

Now press enter twice to skip additional packages and kernel choice.

Know it will ask you about network configurations press 1 for network manager
